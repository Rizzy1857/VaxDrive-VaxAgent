## Component Responsibilities

| Component Name | Responsibility | Key Associated File | Phase |
|:---|:---|:---|:---:|
| **Hardware Token** | Derives cryptographic keys directly from physical hardware, bypassing disk storage. | `HardwareTokenProviders.cs` | 9 |
| **VaxAgent** | The core executable daemon. Wraps all checks, orchestrates timing, and handles service lifetimes. | `AgentBootstrap.cs` | 1 |
| **VaxDock** | The shared data layer. Handles SQLCipher connections and offline CVE intelligence. | `CveRepository.cs` | 1 |
| **TopologyMap** | Passively listens to network traffic (DNP3, CIP, BACnet) to map the factory floor layout. | `TopologyMap.cs` | 9 |
| **YaraScanner** | Reads running process memory in 4MB chunks to match OT-specific malware signatures. | `ProcessMemoryReader.cs` | 9 |
| **NvdPaginationEngine** | Caches the NVD database incrementally and computes severity deltas for discovered assets. | `NvdPaginationEngine.cs` | 9 |
| **AlertDispatcher** | Formats findings into ArcSight/Splunk compatible CEF strings and writes to safe local sinks. | `AlertDispatcher.cs` | 11 |
| **ReportExporter** | Generates completely offline, HMAC-signed HTML and JSON artifacts of all findings. | `ReportExporter.cs` | 11 |
| **OperatorConsole** | A read-only Text User Interface providing live statistics without blocking background tasks. | `OperatorConsole.cs` | 11 |

## Security Boundaries

1. **Memory Zeroization:** Cryptographic material fetched from the hardware token is pinned in unmanaged memory, transferred to a `SecureString`, and instantly wiped using `Array.Clear` and `Marshal.ZeroFreeBSTR`.
2. **Encrypted Storage:** All cached CVE intelligence is stored inside a `.vaxdb` SQLite database secured by SQLCipher using AES-256 keys derived from the hardware token.
3. **Signed Outputs:** Every HTML report, JSON blob, and flat-file log generated by the agent is signed with an `HMAC-SHA256` signature using the build key, preventing stealth tampering by an attacker.
